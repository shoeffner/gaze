151  /**
152  * Calculates the intersection between the line going through
153  * eye_ball_center and pupil and the screen plane.
154  *
155  * The three screen points should describe the plane (i.e. don't lie in one
156  * line).
157  *
158  * This method solves the following system (see
159  * <a href="https://en.wikipedia.org/wiki/Line%E2%80%93plane_intersection#Parametric_form">wikipedia for details</a>):
160  * @f[
161  * \left(\begin{array}{c}
162  * t \\
163  * u \\
164  * v
165  * \end{array}\right)
166  * =
167  * \left(\begin{array}{ccc}
168  * e_x - p_x & b_x - a_x & c_x - a_x \\
169  * e_y - p_y & b_y - a_y & c_y - a_y \\
170  * e_z - p_z & b_z - a_z & c_z - a_z
171  * \end{array}\right)^{-1}
172  * \left(\begin{array}{c}
173  * e_x - a_x \\
174  * e_y - a_y \\
175  * e_z - a_z
176  * \end{array}\right),
177  * @f]
178  * with @f$a, b, c@f$ being the screen corners, @f$e, p@f$ the eye ball
179  * center and pupil, all in model coordinates.
180  *
181  * @param eye_ball_center The eye ball center
182  * @param pupil The pupil
183  * @param screen_a one screen point
184  * @param screen_b one screen point
185  * @param screen_c one screen point
186  *
187  * @returns A matrix containing two column vectors:
188  * The first column is the 3D gaze point, the second
189  * column contains the scalars t, u, v.
190  */
191  cv::Matx32d calculate_gaze_point(
192  const cv::Vec3d& eye_ball_center, const cv::Vec3d& pupil,
193  const cv::Vec3d& screen_a, const cv::Vec3d& screen_b,
